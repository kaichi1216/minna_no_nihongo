# Flashcard 單字互動訓練 — 設計文件

## 概述

在《みんなの日本語》學習網站的每一課中，加入 Flashcard 翻牌式單字訓練功能。使用者點擊「開始練習」按鈕後，單字列表隱藏，原地展開翻牌訓練區域。練習結果透過 localStorage 保存，支援重複練習不認識的單字。

## 功能需求

### 入口

- 每課的「単語」標題上方新增一個「開始練習」按鈕（紅色圓角按鈕）
- 按鈕旁顯示上次練習成績（例如「上次：22/35 認識」），若無記錄則不顯示
- 點擊按鈕後：該課 `.lesson` 內除了 `.lesson-header` 和練習區域以外的所有內容隱藏，展開 Flashcard 訓練區域

### Flashcard 訓練區

**頂部控制列：**
- 課名標示（如「第1課 練習」）
- 當前方向標示（「日文 → 中文」或「中文 → 日文」）
- 「切換方向」按鈕（切換後從下一張卡片開始套用新方向，當前卡片不受影響）
- 「結束」按鈕（✕）— 不儲存部分進度，直接回到單字列表

**進度條：**
- 水平進度條，顯示已完成比例
- 文字標示「第 N / M 張」

**單字卡：**
- 大張卡片居中顯示
- 正面（未翻牌）：
  - 日→中模式：顯示假名（大字）+ 漢字（小字，若有）
  - 中→日模式：顯示中文翻譯
- 點擊卡片翻牌，顯示背面：
  - 日→中模式：顯示中文翻譯
  - 中→日模式：顯示假名 + 漢字
- 翻牌動畫：簡單的 CSS transition（opacity 或 transform）

**翻牌後操作：**
- 顯示兩個按鈕：「不認識」（紅色）和「認識」（綠色）
- 點擊後自動進入下一張卡片
- 未翻牌時不顯示這兩個按鈕

### 結束畫面

- 顯示「練習完成！」
- 統計數據：認識數、不認識數、正確率（百分比）
- 兩個操作按鈕：
  - 「只練不認識的 (N)」— 重新開始，只包含標記為不認識的單字
  - 「回到單字列表」— 收起訓練區域，恢復原本的單字列表顯示

### 出題規則

- 每次開始練習時，將該課全部單字以 Fisher-Yates 演算法隨機洗牌
- 「只練不認識的」也會隨機洗牌
- 所有 `.vocab-row` 均納入練習，包含片語型條目（如「どうぞ よろしく」）

## 資料設計

### 資料來源

直接從 DOM 中讀取。每個 `.vocab-row` 包含：
- `.word` — 假名
- `.kanji` — 漢字（可能為空）
- `.translation` — 中文翻譯
- `.speak-btn[data-text]` — 語音朗讀文字

啟動練習時，用 JavaScript 遍歷該課 `.lesson` 區域內的所有 `.vocab-row`，抽取資料組成陣列。

### localStorage 儲存

Key 格式：`minna-practice-lessonN`（N 從 `.lesson` 元素的 `id` 屬性取得，例如 `id="lesson26"` → `minna-practice-lesson26`）

Value 格式（JSON）：
```json
{
  "lastDate": "2026-03-18",
  "words": {
    "わたし|我": true,
    "せんせい|老師": false,
    "がくせい|學生": true
  }
}
```

- `lastDate`：最後一次練習日期（ISO 格式日期字串）
- `words`：以 `假名|翻譯` 作為複合 key（避免同課內假名重複導致資料覆蓋），`true` 代表認識，`false` 代表不認識
- 「只練不認識的」完成後，將結果合併回完整單字記錄（更新被重練的單字狀態，保留未重練的單字狀態），再覆寫 localStorage
- `localStorage.setItem` 若拋出 `QuotaExceededError`，靜默失敗，不影響練習體驗

## UI 設計

### 視覺風格

延續現有網站風格：
- 背景色：`#fdf8f0`
- 主色調：`#c0392b`（紅色）
- 卡片：白色背景 + `#e8ddd0` 邊框 + 圓角
- 字體：繼承現有 font-family

### 新增 CSS

- `.practice-btn` — 開始練習按鈕樣式
- `.practice-area` — 訓練區域容器
- `.practice-header` — 頂部控制列
- `.practice-progress` — 進度條
- `.flashcard` — 單字卡樣式
- `.flashcard.flipped` — 翻牌狀態
- `.practice-actions` — 認識/不認識按鈕
- `.practice-result` — 結束畫面

### 響應式設計

- 桌面：卡片寬度固定（約 400px），水平置中
- 手機：卡片寬度 100%，按鈕全寬，觸控友善的間距

## 技術實作

### 修改範圍

修改兩個檔案（內容相同）：
1. `shokyuu1/index.html` — 初級 I（第1-25課）
2. `shokyuu2/index.html` — 初級 II（第26-50課）

### 修改內容

1. **CSS**（加在 `</style>` 之前）：Flashcard 相關的所有樣式
2. **HTML**（每課 `.lesson` 區域）：在單字區段上方加入練習按鈕的 HTML
3. **JavaScript**（加在 `</script>` 之前）：
   - 練習按鈕的事件處理
   - Flashcard 區域的動態生成
   - 翻牌、認識/不認識邏輯
   - localStorage 讀寫
   - 結束畫面與重練邏輯

### 不修改的部分

- 不新增任何檔案
- 不修改現有的單字列表、文型卡片、搜尋、語音朗讀等功能
- 不修改側邊欄、導覽等結構
- 不加入鍵盤快捷鍵（未來可作為增強功能）
- Flashcard 訓練不與現有的「顯示/隱藏全部翻譯」功能互動
